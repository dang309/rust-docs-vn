# Định nghĩa

Struct trong Rust cũng tương tự struct trong C++, dictionary trong Python hay object trong JavaScript đó là định nghĩa 1 đối tượng 
với những thông tin miêu tả đối tượng đó.

Để định nghĩa 1 struct trong Rust, sử dụng từ khóa `struct`, sau đó là tên của struct. Bên trong struct chúng ta định nghĩa tên và kiểu dữ liệu

Ví dụ:

```rust title="src/main.rs" showLineNumbers
struct User {
    active: bool,
    username: String,
    email: String,
    sign_in_count: u64,
}

```

## Khởi tạo

Để sử dụng 1 struct, ta khởi tạo 1 đối tượng(instance) với các giá trị cho các trường(fields)

Ví dụ:

```rust title="src/main.rs" showLineNumbers
fn main() {
    let user1 = User {
        active: true,
        username: String::from("someusername123"),
        email: String::from("someone@example.com"),
        sign_in_count: 1,
    };
}
```

## Getter và Setter

Để lấy 1 giá trị từ struct sử dụng ký hiệu `.`(dot). Ví dụ, muốn lấy giá trị của trường `email`, sử dụng `user1.email`. Nếu đổi tượng là `mutable`(Chương 3.1) thì có thể thay đổi dữ liệu bằng cách gán cho trường đó giá trị mới. Ví dụ về việc thay đổi giá trị của `email`:

```rust title="src/main.rs" showLineNumbers
fn main() {
    let mut user1 = User {
        active: true,
        username: String::from("someusername123"),
        email: String::from("someone@example.com"),
        sign_in_count: 1,
    };

    user1.email = String::from("anotheremail@example.com");
}
```

:::note
Toàn bộ instance phải là `mutable` chứ không thể chỉ `mutable` 1 vài fields.
:::

Chúng ta cũng có thể return 1 instance từ 1 function

```rust title="src/main.rs" showLineNumbers
fn build_user(email: String, username: String) -> User {
    User {
        active: true,
        username: username,
        email: email,
        sign_in_count: 1,
    }
}
```

```js showLineNumbers
function HighlightSomeText(highlight) {
  if (highlight) {
    // highlight-next-line
    return 'This text is highlighted!';
  }

  return 'Nothing highlighted';
}

function HighlightMoreText(highlight) {
  // highlight-start
  if (highlight) {
    return 'This range is highlighted!';
  }
  // highlight-end

  return 'Nothing highlighted';
}
```

:::note
Trong trường hợp tên field trùng với tên parameter(`username` và `email`) thì chúng ta có thể rút gọn với `Field Init Shorthand`:

```js showLineNumbers
function HighlightSomeText(highlight) {
  if (highlight) {
    // highlight-next-line
    return 'This text is highlighted!';
  }

  return 'Nothing highlighted';
}

function HighlightMoreText(highlight) {
  // highlight-start
  if (highlight) {
    return 'This range is highlighted!';
  }
  // highlight-end

  return 'Nothing highlighted';
}
```

```rust title="src/main.rs" showLineNumbers
fn build_user(email: String, username: String) -> User {
    User {
        active: true,
        highlight-next-line
        username: username,
        // highlight-next-line
        email: email,
        sign_in_count: 1,
    }
}
```
:::


## Getting Started

Get started by **creating a new site**.

Or **try Docusaurus immediately** with **[docusaurus.new](https://docusaurus.new)**.

### What you'll need

- [Node.js](https://nodejs.org/en/download/) version 16.14 or above:
  - When installing Node.js, you are recommended to check all checkboxes related to dependencies.

## Generate a new site

Generate a new Docusaurus site using the **classic template**.

The classic template will automatically be added to your project after you run the command:

```bash
npm init docusaurus@latest my-website classic
```

You can type this command into Command Prompt, Powershell, Terminal, or any other integrated terminal of your code editor.

The command also installs all necessary dependencies you need to run Docusaurus.

## Start your site

Run the development server:

```bash
cd my-website
npm run start
```

The `cd` command changes the directory you're working with. In order to work with your newly created Docusaurus site, you'll need to navigate the terminal there.

The `npm run start` command builds your website locally and serves it through a development server, ready for you to view at http://localhost:3000/.

Open `docs/intro.md` (this page) and edit some lines: the site **reloads automatically** and displays your changes.
